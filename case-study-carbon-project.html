Carbon Project Australia:
an R&DTI software business,
run on the harness.

"We eat our own cooking" is easy to say. This is the receipt: Carbon Project Australia — our R&D tax software business — runs its evidence capture, document intelligence and agent workflows on the same systems we install for clients, live in production for real claims. Below is what we deployed, how we measured it, and the numbers.

Publication note: every figure on this page is measured before it is published. Values marked MEASURE are pending final verification against our own ledgers and bills — the same standard we apply before claiming anything to a client.
Context

The business and the problem.

R&DTI work is document-heavy and evidence-critical: client records arrive as timesheets, invoices, lab notebooks and project reports; claims live or die on contemporaneous substantiation; and every engagement repeats the same triage → extract → cross-reference → draft cycle. Exactly the shape of work per-seat AI tools meter badly — high volume, all day, across sensitive client data that should never sit in a third-party cloud.

What runs on the box

Three workloads, one harness.

Evidence capture

Client records are ingested and graphed into the knowledge graph — entities, projects, activities and expenditures cross-referenced automatically, with gaps flagged for a human.

Document intelligence

Timesheets, invoices and technical reports are read, classified and reconciled by local models — no client document leaves the premises.

Agent workflows

Draft activity assessments, expenditure schedules and technical narratives are staged by agents overnight and presented for professional review — GREEN actions run, consequential steps wait for sign-off.

How we measure

Methodology — the same one we'll apply to you.

Baseline first

Before-state figures come from time records and supplier bills over a defined baseline window, not memory. Where a clean baseline doesn't exist, we say so rather than estimate one.

Ledger, not vibes

On-harness figures are read from the audit ledger — run counts, timestamps, token telemetry, egress events. The same evidence pack a client (or auditor) can demand.

Like for like

Cost comparisons hold scope constant: the same workloads, the same period length, no mixing of one-off project costs into run-rate.

What we don't claim

Local tokens aren't frontier tokens 1:1, and capacity isn't automatically productivity. We publish what was measured and let the numbers carry the argument.
